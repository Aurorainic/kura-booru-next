> 状态：规划中
> 日期：2026-06-25

---

## 背景

目前所有配置都靠 `.env` 环境变量，改任何东西都要编辑文件再重启容器。需要一个 DB 驱动的设置系统 + 管理后台 UI，支持：

- 站点标题 / 描述（可从管理后台直接改）
- 站点公告（进入网站时从右上角弹出 Toast，支持 Markdown，5s 倒计时条自动收回）
- Head 注入（接入 Umami 等访问追踪系统）
- PG/Redis 连接配置（修改前必须通过连通性测试，保存后下次重启生效）

`.env` 中已有的值在首次启动时自动初始化到 DB，之后 DB 值优先。

## 一、数据库模型

**新建文件**：`backend/app/models/setting.py`

简单 key-value 表，`key` 为主键（自然标识符）。不用 UUID，不加 category 列（分类是前端 UI 的事）。

```python
class Setting(Base):
    __tablename__ = "settings"

    key: Mapped[str] = mapped_column(String, primary_key=True)
    value: Mapped[str] = mapped_column(Text, nullable=False, server_default="")
    updated_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True),
        server_default=func.now(),
        onupdate=func.now(),
        nullable=False,
    )
```

### 设置项列表（6 个，YAGNI）

| Key | 默认值 | 类型 | 分类 |
|-----|--------|------|------|
| `site_title` | `"Kura Booru"` | str | 通用 |
| `site_description` | `"个人动漫插画收藏与展示平台"` | str | 通用 |
| `announcement` | `""` | str (Markdown) | 通用 |
| `head_inject` | `""` | str (原始 HTML) | 通用 |
| `database_url` | 从 `DATABASE_URL` 环境变量 | str | 基础设施 |
| `redis_url` | 从 `REDIS_URL` 环境变量 | str | 基础设施 |

### 需注册的位置

- `backend/app/models/__init__.py` — `from app.models.setting import Setting`，加入 `__all__`
- `backend/alembic/env.py` — import 行加入 `Setting`

## 三、Service 层

**新建文件**：`backend/app/services/settings.py`

### Redis 缓存策略

单 hash key `kura:settings`，一次 `HGETALL` 全量读取。TTL 300s。

更新时：`HMSET` 刷新缓存 + 重设 TTL。

### 函数清单

```python
async def get_all_settings(db: AsyncSession) -> dict[str, str]:
    """加载全部设置。Redis 优先，DB 兜底，miss 时填充缓存。"""

async def get_setting(db: AsyncSession, key: str, default: str = "") -> str:
    """获取单个设置值。"""

async def update_settings(db: AsyncSession, updates: dict[str, str]) -> dict[str, str]:
    """批量更新设置。INSERT ... ON CONFLICT DO UPDATE upsert，更新后刷新缓存。"""

async def test_pg_connectivity(url: str) -> ConnectivityTestResponse:
    """测试 PG 连通性。创建临时引擎，SELECT 1，销毁。不影响运行中的引擎。"""

async def test_redis_connectivity(url: str) -> ConnectivityTestResponse:
    """测试 Redis 连通性。创建临时客户端，PING，关闭。不影响运行中的客户端。"""

async def seed_settings_from_env(db: AsyncSession) -> None:
    """从环境变量填充 DB 中不存在的 key。INSERT ... ON CONFLICT DO NOTHING。"""
```

### Redis 客户端

复用 `auth.py` 的 `_get_redis()` 单例，不重复创建。

### 连通性测试安全设计

- 创建**临时**连接（新引擎/新客户端），测试后立即销毁
- 绝不触碰运行中的引擎或 Redis 客户端
- 测试失败 → 运行中的服务完全不受影响

### PG/Redis 修改生效方式

**保存到 DB，下次容器重启时生效。** 运行中的连接不会断开，不做热重载。

原因：
- 热重载需要重建引擎/客户端，有短暂断连风险
- 连接池中的活跃事务可能失败
- 重启生效是最安全最简单的方案

## 四、API 端点

**新建文件**：`backend/app/api/settings.py`，前缀 `/api/settings`

### 端点列表

| 方法 | 路径 | 鉴权 | 说明 |
|------|------|------|------|
| GET | `/` | `get_current_admin` | 全部设置（含基础设施 URL） |
| PUT | `/` | `get_current_admin` | 批量更新 `{"settings": {"key": "value", ...}}` |
| GET | `/public` | 无 | 仅非敏感设置（site_title, site_description, announcement, head_inject） |
| POST | `/test-pg` | `get_current_admin` | 测试 PG 连通性 `{"url": "..."}` |
| POST | `/test-redis` | `get_current_admin` | 测试 Redis 连通性 `{"url": "..."}` |

### Schema 定义

Schema 定义在路由文件内（跟 `auth.py`、`tasks.py` 一样的已有模式）。

```python
class SettingRead(BaseModel):
    key: str
    value: str
    updated_at: str  # ISO 格式

class SettingsResponse(BaseModel):
    settings: list[SettingRead]

class SettingsUpdate(BaseModel):
    settings: dict[str, str]  # key → new value

class ConnectivityTestRequest(BaseModel):
    url: str

class ConnectivityTestResponse(BaseModel):
    ok: bool
    detail: str  # "Connected" 或错误信息

class PublicSettingsResponse(BaseModel):
    site_title: str
    site_description: str
    announcement: str
    head_inject: str
```

### 公开端点设计

`GET /api/settings/public`：
- 不需要鉴权
- 读 Redis 缓存（cache hit 不打 DB）
- 返回固定 4 个非敏感 key
- 永远不暴露 `database_url` / `redis_url`

### 注册路由

在 `backend/app/api/__init__.py` 中：

```python
from app.api.settings import router as settings_router
api_router.include_router(settings_router, prefix="/api/settings", tags=["settings"])
```

## 七、BaseLayout 改动

修改 `frontend/src/layouts/BaseLayout.astro`：

### 7a. Head 注入

- `<head>` 内加 `<slot name="head" />`（放在主题检测脚本之后）
- 从 `Astro.locals.siteSettings` 读取 `headInject`
- 渲染方式：`{headInject && <Fragment set:html={headInject} />}`

### 7b. 站点标题

- 从 `siteSettings.site_title` 读取，替代硬编码 `"Kura Booru"`
- 用于 `<title>` 和导航栏文字
- 导航栏标题拆分逻辑：空格前为 gradient-text，空格后为 muted text

### 7c. 公告 Toast

从 `siteSettings.announcement` 读取，非空时渲染。

**视觉设计**：
- 位置：右上角，`fixed top-4 right-4 z-50`
- 最大宽度 `max-w-sm`，卡片样式 + 左边框 accent 色条
- 内容区域 + 关闭按钮（X 图标）
- 底部倒计时条：1s 内从 100% → 0%，5s 线性动画

**交互**：
- 页面加载时自动弹出（无动画延迟，立即可见）
- 倒计时条开始消耗
- 5s 后自动收回（向右滑出 + 透明度渐变，0.3s）
- 点击关闭按钮：取消倒计时，立即收回
- 收回后 DOM 节点移除

**Markdown 渲染**：内联 10 行 `simpleMarkdown()` ES5 函数，处理：
- `**加粗**` → `<strong>`
- `*斜体*` → `<em>`
- `` `代码` `` → `<code>`
- `[链接](url)` → `<a target="_blank">`
- `\n` → `<br>`

不加新依赖。如需更复杂 Markdown，后续再加库。

**脚本注入方式**：`<script define:vars={{ content: announcement }}>` 传递内容。

### 7d. 管理员菜单

在桌面下拉菜单和移动端菜单中，"修改密码"之后新增"站点设置"链接：

```astro
<a href="/admin/settings" class="flex items-center gap-2.5 px-3 py-2 ...">
  <!-- 齿轮图标 SVG -->
  站点设置
</a>
```

## 八、管理后台设置页

**新建文件**：`frontend/src/pages/admin/settings.astro`

沿用现有管理员页面模式：`BaseLayout`、`isAdmin` 守卫、SSR fetch 初始数据、内联 ES5 脚本。

### 页面布局

三个卡片区域：

**1. 通用**
- 站点标题（input text）
- 站点描述（input text）
- 公告内容（textarea，4 行，下方提示"支持 **加粗**、*斜体*、`代码`、[链接](url)"）
- Head 注入（textarea，6 行，等宽字体，下方提示"用于注入分析脚本等 HTML 内容"）

**2. 基础设施**
- 数据库 URL（input text）+ "测试连接" 按钮
- Redis URL（input text）+ "测试连接" 按钮
- 提示文字："修改后下次重启生效"

**3. 保存按钮**

### 基础设施字段行为

- 初始加载时，记录 `database_url` 和 `redis_url` 的原始值
- 输入值变化时，"保存"按钮禁用，对应字段旁显示"已修改"标记
- 必须先点"测试连接"并成功（绿色 ✓），才能保存
- 测试按钮交互：
  - 点击后显示 loading spinner
  - 成功：按钮变绿 ✓，"保存"按钮中该字段变为可保存
  - 失败：按钮变红 ✗ + 错误提示
- 保存时，仅将通过测试的基础设施字段包含在 PUT 请求中
