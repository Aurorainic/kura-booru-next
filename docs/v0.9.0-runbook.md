### 0.4 代码规则（违反 = 返工）

1. **新代码全部显式 import，禁用 Nitro auto-import**。原因：backend-audit §8.2-1 实录 auto-import 故障（bot-rating.ts 的 `redis$1` bug）。路由文件里只允许 `defineEventHandler` 之外不出现裸用 `db/redis/posts/...`。
2. **相对路径深度**：`server/routes/api/<group>/x.ts` → `server/` 是 `../../../`（3 层）；`api/<group>/<sub>/x.ts` → 4 层；`api/<group>/<sub>/<sub2>/x.ts` → 5 层。写错会报 TS2307。
3. **错误一律 `AppError`（`server/platform/errors.ts`）**，输出形状 `{ code, message, details? }`。旧代码里的 `throw createError(...)` 迁移时改为 AppError；`code` 用 SNAKE_CASE。
4. **不可变契约**（backend-audit §6，动了 = 事故）：
   - `web-import.post.ts`：错误码字面量 `key_not_authorized_for_force_rating`（:73）、`kb_ext_` 前缀、per-key 60/min、响应 `{results:[{task_id,status,url,error?}]}`；
   - `tasks/[id].get.ts`：状态字面量 `queued/in_progress/complete/duplicate/too_large/failed`（:14-22）、响应剥离 `image_bytes_b64/phash`；
   - `bot/webhook.post.ts`：secret_token 校验 + callback_data 协议（`rate:`/`post:`/`random:another`）；
   - `routes/i/[...].ts`：Range 透传 + 流式 + `max-age=31536000` 不加 immutable。
5. **zod 枚举只从 `server/platform/schemas/enums.ts` 取**（zRating/zSourceSite/zTagCategory），消灭手写数组。
6. **注释里的 ponytail/决策记录必须随代码一起搬迁**（尤其 pipeline.ts 的 series 并发注释、dashboard 的 MV 注释）——那是修复史，不是废话。

### 0.5 新范式模板（路由迁移的标准动作）

迁移一个路由 = 把"鉴权 → 手写校验 → 业务"改为"包装 + zod schema + 纯业务"。三个已完成范例，照抄结构：

- session 端点：`server/routes/api/admin/settings/index.get.ts`（defineAdminHandler）
- public 端点：`server/routes/api/settings/public.get.ts`（definePublicHandler）
- session-or-apikey 端点：`server/routes/api/admin/dashboard/index.get.ts`（defineApiKeyHandler，auditAction 必填）

模板（以 admin session + body 校验为例）：

```ts
import { z } from 'zod'
import { defineAdminHandler } from '<深度>/platform/http/auth'
import { zRating } from '<深度>/platform/schemas/enums'

const bodySchema = z.object({ rating: zRating }) // 入参契约

export default defineAdminHandler({
  schemas: { body: bodySchema },
  doc: { method: 'patch', path: '/api/posts/:id', summary: 'Update post rating' },
  handler: async ({ body, params }) => { /* 纯业务，body 已校验且类型化 */ },
})
```

注意：query 参数值都是字符串，schema 用 `z.coerce.number()` 等；校验失败自动返回 400 `{ code: 'VALIDATION_FAILED', details: [...] }`。
